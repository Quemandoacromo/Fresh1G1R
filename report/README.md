## Report File Names

Reports retain their original Retool-generated filenames, which include:
- System name
- Original DAT date
- Retool version and timestamp
- Retool flags used

Example: `Sony - PlayStation (2025-10-23 18-11-28) (Retool 2026-01-10 12-39-40) (-l) [-aAbBcdDefkmMopPruv] report.txt`

## Report Retention

- The **last 7 reports** per system are kept automatically
- Older reports are cleaned up to save space
- This ensures you can review recent processing history without accumulating too many files

## How to Use Reports

1. **Review Filtering**: See exactly what was kept/removed for each system
2. **Troubleshooting**: Understand why certain ROMs were filtered out
3. **Verification**: Verify that the filtering rules are working as expected
4. **Statistics**: Check processing statistics and counts

## Understanding Report Format

Reports use a simple format:
- `+` prefix = ROM/game was **kept** (included)
- `-` prefix = ROM/game was **removed** (filtered out)
- Indentation shows relationships (e.g., which version was kept vs. removed)
